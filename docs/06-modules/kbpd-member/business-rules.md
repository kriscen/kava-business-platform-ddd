# kbpd-member -- 业务规则

> **当前状态：** 会员模块处于早期脚手架阶段，领域层为空，尚未实现任何业务规则。以下为基于模块定位的业务规则规划，待领域建模时细化和实现。

---

## 已确认的设计约束

| 约束 | 说明 |
|------|------|
| 多租户隔离 | 所有查询需携带 `tenantId` 参数，数据按租户隔离 |
| CQRS 读写分离 | 仓储接口将拆分为 `ReadRepository` 和 `WriteRepository` |
| DDD 分层隔离 | 业务规则必须位于 domain 层，不得泄露到 application 或 adapter 层 |
| 领域层无框架依赖 | domain 层不得依赖 Spring 或其他外部框架 |

---

## 预期业务规则（规划）

### 会员管理

| 规则 | 说明 | 优先级 |
|------|------|--------|
| 手机号唯一性 | 同一租户内，一个手机号只能注册一个会员 | 高 |
| 会员状态管理 | 会员可被启用/禁用，禁用后不可登录和使用服务 | 高 |
| 会员注册 | 需校验手机号格式和唯一性 | 高 |
| 会员信息更新 | 允许更新基本信息，手机号变更需验证 | 中 |

### 角色与权限

| 规则 | 说明 | 优先级 |
|------|------|--------|
| 角色分配 | 会员可被分配一个或多个角色 | 高 |
| 权限继承 | 会员的权限由其角色决定，不直接分配权限 | 中 |
| 权限查询 | 支持通过 RPC 查询会员的角色和权限列表 | 高 |

### 数据安全

| 规则 | 说明 | 优先级 |
|------|------|--------|
| 敏感信息保护 | 手机号等敏感信息存储和传输时需脱敏处理 | 中 |
| 租户数据隔离 | 任何数据访问必须限定在当前租户范围内 | 高 |

---

## 开发优先级建议

1. **领域建模** — 定义 `Member` 聚合根、值对象（手机号、会员状态等）
2. **仓储实现** — 建表、实现 MyBatis Mapper 和 PO 转换
3. **核心查询** — 将 RPC 桩方法替换为真实查询逻辑
4. **注册与状态管理** — 实现会员注册、状态变更等核心业务
5. **HTTP 接口** — 基于已验证的领域模型暴露 REST API

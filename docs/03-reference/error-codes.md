# 错误码规范

## 5.1 错误码结构

### 5.1.1 错误码格式

```
{系统前缀}-{模块编号}-{具体编号}

示例：
UPMS-001-001  系统用户不存在
AUTH-002-003  Token 已过期
```

### 5.1.2 系统前缀

| 前缀 | 服务/系统 | 说明 |
|------|----------|------|
| `UPMS` | 用户权限管理系统 | kbpd-upms |
| `AUTH` | 认证服务 | kbpd-auth |
| `MEMBER` | 会员服务 | kbpd-member |
| `GATEWAY` | 网关服务 | kbpd-gateway |
| `SYS` | 系统级错误 | 通用 |

### 5.1.3 错误级别

| 级别 | 值 | 说明 |
|------|-----|------|
| `FATAL` | 1 | 系统级严重错误，需紧急处理 |
| `ERROR` | 2 | 业务执行失败，需要排查 |
| `WARN` | 3 | 警告，可能存在问题 |
| `INFO` | 4 | 信息，非错误 |

## 5.2 通用错误码

### 5.2.1 系统级错误（0xx）

| 错误码 | 错误信息 | HTTP Status | 说明 |
|--------|---------|-------------|------|
| `SYS-000-001` | 系统内部错误 | 500 | 未预期的服务器错误 |
| `SYS-000-002` | 服务不可用 | 503 | 服务降级或维护中 |
| `SYS-000-003` | 请求超时 | 504 | Gateway 超时 |
| `SYS-000-004` | 参数校验失败 | 400 | 通用参数校验错误 |
| `SYS-000-005` | 缺少必需参数 | 400 | 必填参数为空 |
| `SYS-000-006` | 无效的请求格式 | 400 | JSON 解析失败等 |
| `SYS-000-007` | 资源不存在 | 404 | 路径或资源不存在 |
| `SYS-000-008` | 请求方法不支持 | 405 | PUT/DELETE 等不支持 |

### 5.2.2 认证授权错误（1xx）

| 错误码 | 错误信息 | HTTP Status | 说明 |
|--------|---------|-------------|------|
| `SYS-001-001` | 未提供认证信息 | 401 | 缺少 Authorization Header |
| `SYS-001-002` | 无效的认证信息 | 401 | Token 格式错误 |
| `SYS-001-003` | Token 已过期 | 401 | access_token 过期 |
| `SYS-001-004` | RefreshToken 已过期 | 401 | refresh_token 过期 |
| `SYS-001-005` | 无访问权限 | 403 | 缺少必要权限 |
| `SYS-001-006` | 账号已被锁定 | 403 | 用户被锁定 |
| `SYS-001-007` | 账号已被禁用 | 403 | 租户或用户被禁用 |
| `SYS-001-008` | 签名验证失败 | 401 | 接口签名错误 |

### 5.2.3 租户错误（2xx）

| 错误码 | 错误信息 | HTTP Status | 说明 |
|--------|---------|-------------|------|
| `SYS-002-001` | 缺少租户信息 | 400 | 未提供 X-Tenant-Id |
| `SYS-002-002` | 租户不存在 | 404 | 指定租户 ID 不存在 |
| `SYS-002-003` | 租户已禁用 | 403 | 租户状态非正常 |

| `UPMS-100-005` | 原密码错误 | 修改密码时原密码不正确 |

| `UPMS-100-007` | 用户已锁定，无法操作 | 锁定状态用户禁止部分操作 |
| `UPMS-100-008` | 用户已删除 | 软删除用户禁止操作 |

## 5.6 错误响应格式

### 5.6.1 标准错误响应

```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

当 `code != 200` 时，表示业务错误：

```json
{
  "code": 400,
  "message": "用户名已存在",
  "errorCode": "UPMS-100-002",
  "timestamp": "2024-01-15T10:30:00Z",
  "requestId": "req-uuid-12345",
  "traceId": "trace-uuid-67890"
}
```

### 5.6.2 参数校验错误响应

```json
{
  "code": 400,
  "message": "参数校验失败",
  "errorCode": "SYS-000-004",
  "errors": [
    {
      "field": "password",
      "message": "密码长度不能少于6位"
    },
    {
      "field": "email",
      "message": "邮箱格式不正确"
    }
  ]
}
```

### 5.6.3 全局异常映射

| 异常类型 | HTTP Status | Error Code |
|---------|-------------|------------|
| `DomainException` | 400 | 业务错误码 |
| `IllegalArgumentException` | 400 | `SYS-000-004` |
| `AuthenticationException` | 401 | `SYS-001-xxx` |
| `AccessDeniedException` | 403 | `SYS-001-005` |
| `ResourceNotFoundException` | 404 | 业务资源错误码 |
| `MethodArgumentNotValidException` | 400 | `SYS-000-004` |
| `ConstraintViolationException` | 400 | `SYS-000-004` |

## 5.7 错误处理规范

### 5.7.1 错误抛出原则

```java
// ✓ 正确：使用业务错误码
throw new UserNotFoundException(userId);

// ✗ 错误：直接抛出通用异常
throw new RuntimeException("用户不存在");
```

### 5.7.2 错误码定义示例

```java
// 错误码枚举
public enum ErrorCode {
    // 系统级
    SYSTEM_ERROR("SYS-000-001", "系统内部错误", 5),

    // 用户模块
    USER_NOT_FOUND("UPMS-100-001", "用户不存在", 4),
    USERNAME_EXISTS("UPMS-100-002", "用户名已存在", 4),

    // 认证模块
    AUTH_FAILED("AUTH-001-001", "用户名或密码错误", 4),
    TOKEN_EXPIRED("SYS-001-003", "Token 已过期", 4);

    private final String code;
    private final String message;
    private final int level;  // 1=FATAL, 2=ERROR, 3=WARN, 4=INFO

    ErrorCode(String code, String message, int level) {
        this.code = code;
        this.message = message;
        this.level = level;
    }
}

// 自定义异常
public class UserNotFoundException extends BusinessException {
    public UserNotFoundException(Long userId) {
        super(ErrorCode.USER_NOT_FOUND, "userId: " + userId);
    }
}
```

### 5.7.3 日志记录规范

```java
// ✗ 错误：ERROR 级别但包含敏感信息
log.error("登录失败: password={}", password);

// ✓ 正确：WARN 级别，不记录敏感信息
log.warn("登录失败: username={}", username);

// ✓ 正确：记录异常堆栈（生产环境脱敏）
log.error("用户创建异常: userId={}", userId, e);
```

# 用户角色定义

## 角色总览

Kava 平台的用户分为两大类：**B端**（管理端）和 **C端**（应用端）。

```
Kava 用户
├── B端（管理端）
│   ├── 平台管理员（Platform Admin）
│   └── 租户管理员（Tenant Admin / 家庭管理员）
└── C端（应用端）
    └── App 终端用户（App End User）
```

---

## B端 — 平台管理员

| 属性 | 说明 |
|------|------|
| 定义 | 管理整个 Kava 平台的超管角色 |
| 归属 | 不属于任何特定租户，处于租户体系之上 |
| 主要职责 | 平台全局配置、租户生命周期管理、系统级监控与运维 |
| 权限边界 | 可查看和管理所有租户的元数据（如租户列表、状态），但不直接操作租户内部业务数据 |
| 使用入口 | Kava 平台管理后台 |

### 典型场景

- 开通/停用/删除租户
- 配置平台级参数（如默认功能开关、全局策略）
- 监控平台运行状态
- 处理跨租户的系统级问题

---

## B端 — 租户管理员（家庭管理员）

| 属性 | 说明 |
|------|------|
| 定义 | 管理单个家庭（租户）的管理员角色 |
| 归属 | 属于特定租户，数据严格限制在租户范围内 |
| 主要职责 | 家庭成员管理、家庭配置、App 接入管理、家庭数据管理 |
| 权限边界 | 只能操作所属租户的数据，无法感知其他租户的存在 |
| 使用入口 | Kava 家庭管理后台 |

### 典型场景

- 邀请/移除家庭成员
- 设置家庭成员角色和权限
- 注册和管理家庭 App 的 OAuth2 客户端
- 查看家庭数据报表
- 配置家庭级功能参数

---

## C端 — App 终端用户

| 属性 | 说明 |
|------|------|
| 定义 | 通过家庭 App 注册和使用的终端用户 |
| 归属 | 通过 App（OAuth2 Client）关联到特定租户（家庭），用户数据以 Member 形式存在于该家庭中 |
| 注册方式 | 开放注册，通过各家庭 App 的注册入口完成 |
| 使用入口 | 各家庭独立开发的垂直 App |
| 用户类型 | `userType=2`（TO_C），由 App 的 OAuth2 Client 配置决定 |

### 典型场景

- 在家庭 App 中注册/登录（通过 App 对应的 OAuth2 Client 完成认证）
- 使用 App 提供的垂直功能（相册、通讯录、活动等）
- 个人信息管理
- 用户数据作为 Member 存在于 App 归属的家庭（租户）中

## 关键约束

1. **单账号单租户**：一个用户只能属于一个家庭（租户）
2. **强数据隔离**：租户间数据不可互通，所有查询必须限定租户维度
3. **C端不直连后台**：C端用户通过 OAuth2 接入，不直接访问 Kava 管理后台
4. **平台管理员超然**：平台管理员不属于任何租户，管理的是平台层面而非租户内部

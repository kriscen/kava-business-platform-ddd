## 租户模型

### 核心概念

- **租户（Tenant）= 家庭（Family）**：平台的基本组织单位
- 每个租户拥有独立的：成员列表、权限配置、App 注册信息、业务数据
- 租户创建后由租户管理员（家庭管理员）管理

### 租户生命周期

```
创建租户 → 配置家庭信息 → 邀请成员 → 注册 App → 日常使用
                                                      ↓
                                            停用/冻结/删除
```

### 数据隔离策略

| 策略 | 说明 |
|------|------|
| 隔离方式 | 共享数据库 + 租户字段过滤（tenant_id） |
| 查询隔离 | 所有 SQL 查询必须包含 tenant_id 条件 |
| API 隔离 | 请求经过 Gateway 时注入/校验租户上下文 |
| Domain 隔离 | Domain 层的聚合根必须携带租户 ID |
| 超级权限 | 仅平台管理员可跨租户操作，且仅限租户元数据 |

## OAuth2 接入流程

Kava 作为 OAuth2 授权服务器，为各家庭 App 提供统一认证。

### 注册流程

```
家庭管理员                    Kava 后台                     Auth 服务
   │                           │                            │
   │── 注册 OAuth2 客户端 ───→ │                            │
   │                           │── 创建 client_id/secret ──→│
   │                           │                            │
   │← 返回客户端凭证 ──────── │                            │
```

### 用户登录流程

```
App 用户                     App                         Kava Auth
   │                          │                            │
   │── 访问受保护资源 ──────→ │                            │
   │                          │── 302 → /oauth2/authorize ─→│
   │←── 登录页面 ──────────── │                            │
   │── 提交凭据 ─────────────────────────────────────────→ │
   │                          │←── callback(code) ──────── │
   │                          │── POST /oauth2/token ─────→│
   │                          │←── access_token ────────── │
   │←── 返回资源 ──────────── │                            │
```

## 用户模型

### 单账号单租户

```
User 1 ──────→ Tenant A（家庭 A）
User 2 ──────→ Tenant A（家庭 A）
User 3 ──────→ Tenant B（家庭 B）

✗ 不支持：User 1 同时属于 Tenant A 和 Tenant B
```

### 用户与认证方式

| 用户类型 | 认证方式 | Token 特点 |
|---------|---------|-----------|
| 平台管理员 | Kava 后台登录 | 无租户上下文，全局权限 |
| 租户管理员 | Kava 后台登录 | Token 绑定 tenant_id |
| App 终端用户 | OAuth2 授权码流程 | Token 绑定 tenant_id + client_id |

## 模块职责与产品关系

| 模块 | 产品角色 | 当前状态 |
|------|---------|---------|
| kbpd-gateway | 请求路由、租户上下文注入、API 鉴权 | 基础框架就绪 |
| kbpd-auth | OAuth2 授权服务器、统一认证中心 | 多租户认证已支持 |
| kbpd-upms | 用户管理、角色权限、租户管理 | DDD 结构搭建中 |
| kbpd-member | 会员（家庭成员）管理 | DDD 结构搭建中 |
| kbpd-common | 共享基础设施（安全、数据库、缓存等） | 基础能力已具备 |

## 扩展方向

1. **App 市场**：家庭可浏览和接入预制 App 模板
2. **平台级服务**：消息推送、数据分析、文件存储等共享服务
3. **计费系统**：基于租户的订阅或用量计费（商业模式确定后建设）
4. **开放 API**：为 App 开发者提供标准化的平台能力 API

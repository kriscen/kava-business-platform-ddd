# 2024-02-25 限界上下文映射：防腐层（ACL）

## 状态
已采纳

## 背景

微服务划分后，不同限界上下文之间不可避免需要相互调用：

- **Member Context 需要用户信息**：会员服务需要知道持有者的基本信息
- **UPMS Context 需要会员信息**：权限系统可能需要参考会员等级设置权限

如果直接引用对方上下文的对象，会造成**模型污染**和**紧耦合**。

## 决策

跨上下文引用必须通过**防腐层（Anticorruption Layer）**隔离：

```
┌─────────────────────────────────────────────────────────────────┐
│                      Member Context                              │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │              MemberUserAdapter (防腐层)                    │   │
│  │  @DubboReference SysUserFacadeService                     │   │
│  │                                                           │   │
│  │  public MemberUser toMemberUser(SysUserId id) {           │   │
│  │      SysUserDTO dto = sysUserService.getUserById(id);    │   │
│  │      return new MemberUser(                              │   │
│  │          MemberUserId.of(dto.getUserId()),               │   │
│  │          MemberUserName.of(dto.getNickname())            │   │
│  │      );                                                   │   │
│  │  }                                                         │   │
│  └──────────────────────────────────────────────────────────┘   │
│                              │                                  │
│                              ▼                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │              Domain: MemberUser (自有模型)                 │   │
│  │  - MemberUserId                                            │   │
│  │  - MemberUserName                                          │   │
│  │  - 不依赖外部的 SysUserEntity                              │   │
│  └──────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

**上下文映射关系**：

| 源上下文 | 目标上下文 | 映射类型 | 说明 |
|---------|-----------|---------|------|
| Member | UPMS | Conformist（追随） | 会员引用用户，但不需要理解用户创建完整流程 |
| UPMS | Member | Anticorruption（防腐） | UPMS 需要隔离会员相关概念 |

## 理由

1. **隔离模型污染**：自有领域不受外部模型侵蚀
2. **接口稳定**：即使上游服务 API 变化，只需修改适配器
3. **独立演进**：各上下文可独立重构自有模型
4. **可测试**：适配器可以 mock 外部服务，单独测试

## 替代方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| 直接引用对方实体 | 简单 | 模型耦合，无法独立演进 |
| 共享内核（Shared Kernel） | 少一次转换 | 强耦合，牵一发动全身 |
| 开放主机服务（OHS） | 官方推荐 | 实现成本高 |

## 后果

- **正面**：上下文独立演进，不被外部模型绑架
- **正面**：适配器充当缓冲，外部 API 变更影响可控
- **约束**：禁止在领域层直接使用外部上下文的 Entity/DTO
- **约束**：防腐层放在 Application 层或单独 Adapter 模块
